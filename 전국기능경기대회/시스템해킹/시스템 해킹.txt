1. 레지스터를 알고 가기 전, 하드웨어 구성
 CPU에 대한 이해까지 추가적으로 하면 레지스터가 쉬움
 CPU, 메인메모리, 입출력 버스로 구성되어 있다.

 가. CPU란
  - Central Processing Unit의 약자로, 중앙처리장치라고도 불리며 줄로 연산이 이뤄진다.
  - 내부에는 ALU, Control Unit, Bus Interface, Register로 구성되어 있다.
  - ALU(산술 논리 장치) : CPU 내부에서 실제 프로그램이 실행되는 연산을 담당(산술 연산, 논리 연산이 행해짐)
  - Control Unit(제어 장치) : CPU가 처리해야 할 명령어를 해석하고 그에 맞는 신호를 발생
  - Bus Interface(버스 인터페이스) : 컴퓨터 하드웨어 요소들 간의 데이터를 서로 주고 받을 수 있는 통로  *CPU 내부에 데이터가 들어오는 것이 가능한 이유도 이때문
  - Register(레지스터) : CPU 내부의 아주 작은 메모리 공간  *명령어들이 CPU에 들어왔을 때 ALU가 연산 중이고 컨트롤 유닛이 명령어를 해석 중이라면 들어온 나머지 데이터들을 저장해줄 공간 필요. 메인 메모리 외에 CPU 내부에서 이러한 데이터들을 저장하기 위한 아주 작은 메모리가 레지스터

   1. 클럭(Clock) : 컴퓨터는 타이밍에 맞춰 움직임. 전기가 들어오면 1 전기가 들어오지 않으면 0이라는 전기적 신호를 만들어서 데이터 처리.
                   CPU의 클럭이 높다라는 것은 1초에 더 많은 양의 데이터를 처리한다는 의미. 이는 "Hz"라고 표기
 	- Ex) 1Hz=1개의 전기 신호, 4Hz=4개의 전기신호, 1MHz=100만 개의 전기신호, 4GHz=40억 개의 전기 신호
       하지만 신호(클럭)를 많이 처리하는 것이 중요한 것이 아님. 신호를 많이 처리해도 효율이 없으면 아무리 많이 처리해도 느려짐.
       여기서 효율적으로 처리하는 단위가 IPC. 즉 연산능력이 얼마나 효ㅛ율적으로 갖추고 있는지를 말함.
       IPC(Instruction Per Clock) : 명령어를 하나 처리하는데 CPU의 기본 동작인 클러이 얼마나 필요한가를 나타냄
   2. 코어(Core) : 싱글코어=일하는 사람. 빠른 속도로 작업 하나하나 마다 작업한다.
                  멀티코어=하나 이상의 코어, 일하는 사람이 2개 이상으로 늘어남. 그러니깐 싱글코어보다 빠르게 여러작업 수행
   3. 스레드(Thread) : 하나의 물리적인 코어에 2개의 쓰레드로 나눠 마치 물리적인 2개의 코어가 있는 것처럼 동작할 수 있는 것

 나. 메인 메모리
  - 램(RAM:Random Accesss Memory:랜덤 액세스 메모리)이라고 불리는 주기억장치를 의미하며, 프로그램의 코드가 메모리에 적재되어 실행되는 곳이다. 프로그램의 실행을 위해 존재하는 메모리이다.

CPU 캐시메모리

만약 어떤 마을에 식품을 저장하기 위해서 마을과 멀리 떨어져있는 동굴에 보관하는데 동굴에서 왔다갔다 하는 시간이 오래걸려서 조금 더 가까이 있지만 적당히 최근 사람들이 먹는 식품만 보관할 수 있는 동굴을 개설. 하지만 그 마을에 족장이 먹고 싶어하는 식품만을 저장하는 창고를 개설.
-멀리 있지만 많은 식품을 보관할 수 있는 동굴 : 하드디스크/SSD
-조금 더 가까이 위치하지만 적당한 양의 식품만 보관할 수 있는 동굴 : RAM
-마을 내의 위치하지만 조금만 보관할 수 있는 창고 : 캐시 메모리 = 시간적, 공간적 지역성을 기반으로 가까운 미래에 접근될 확률이 높은 데이터를 작지만 빠른 캐시 메모리에 미리 보관하여 전체적인 시스템의 성능을 높인다.

대표적인 캐시 메모리 저장 규칙
 1. 최근에 접근된 데이터
  - Termporal Locality-시간적 지역성
 2. 최근에 접근된 데이터의 주변 데이터
  - Spatial Locality-공간적 지역성
