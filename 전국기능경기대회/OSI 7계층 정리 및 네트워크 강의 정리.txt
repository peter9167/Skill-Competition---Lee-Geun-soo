OSI 7계층 정리

목적 : 계층별로 담당하는 역할을 나누어서 계층에 존재하는 각 프로토콜이 자신의 기능만 충실히 하도록 하여 독립성을 보장하기 위해서

7계층 응용 계층(Application Layer) : 사용자 인터페이스를 제공하는 프로그램 나타냄 ( 프로토콜 : HTTP, FTP )
 * 사람과 사물 또는 시스템, 특히 기계, 컴퓨터 프로그램 등 사이에서 의사소통을 할 수 있도록 일시적 또는 영구적인 접근을 목적으로 만들어진 물리적, 가상적 매개체를 뜻

6계층 표현 계층(Presentation Layer) : 코드 간의 번역, MIME 인코딩이나 암호화 등의 동작

5계층 세션 계층(Session Layer) : 데이터를 만들어내는 계층, 응용 프로세스가 통신을 관리하기 위한 방법을 제공 ( RPC, Socket )
 
4계층 전송 계층(Transport Layer) : End-Device의 사용자들이 데이터를 주고 받을 수 있게 하는 계층 / TCP, UDP 프로토콜
 * TCP : 연결 지향 프로토콜 / 데이터 송수신을 위해 클라이언트와 서버의 소켓이 연결되어 있어야 하며, 데이터가 유실되면 데이터 재전송을 요청함으로써 신뢰성을 보장 / 신뢰성이 있는 데이터 전송이 가능하다는 장점으로 인해 HTTP, FTP, TENET 등의 대부분의 응용 계층 프로토콜의 전송 계층으로 사용됨.
 * UDP : 비연결 지향 프로토콜 / 전송한 데이터가 잘 전달이 되었는지 확인하지 않고 단지 데이터만 보냄 / 신뢰적이지 않으며( 비신뢰성 ), 대신 속도가 빠르다는 장점

3계층 네트워크 계층(Network Layer) : 네트워크 계층에서는 대표적으로 노드들을 거칠때마다 라우팅 해주는 역할을 담당
 * 네트워크 계층에서는 대표적으로 노드 (네트워크에 연결된 장치) / 대표적인 장비는 라우터, L3 스위치, IP 공유기

2계층 데이터 링크 계층(Data Link Layer) : Point to Point 간의 신뢰성 있는 전송을 보장하기 위한 계층 ( 물리주소 : MAC주소)
 * Point to Point : 통신에서 지점 간 연결은 두 통신 끝점 또는 노드 간의 통신 연결을 나타냄. 예로 한 가지 전화가 서로 연결되어 있고 한 발신자가 말한 내용은 다른 발신자 만들을 수있는 전화 통화

1계층 물리 계층(Physical Layer) : 하드웨어 전송 기술, 전기적인, 기계적인 신호를 주고받는 역할을하는 계층, 비트 0, 1의 통신단위로 통신
 * 대표적인 장비는 통신 케이블, 허브, 리피터
